## 引言

本项目是一个基于Hexo框架和Kira主题构建的个人技术博客，旨在为开发者提供一个高效、美观的内容发布平台。博客系统支持文章、页面、友链等多种内容类型，集成了AI助手、评论系统和自动化部署等现代化功能。通过本文档，您将全面了解如何管理博客内容，从创建文章到发布部署的完整流程。

## 博客内容管理流程

博客内容管理涵盖了从文章创作、编辑到发布的完整生命周期。整个流程始于使用Hexo命令创建新文章，然后在Markdown文件中编写内容并填写Front Matter元数据，最后通过构建和部署命令将内容发布到线上服务器。

创建新文章的命令为`hexo new "文章标题"`，该命令会根据`scaffolds/post.md`模板在`source/_posts`目录下生成新的Markdown文件。草稿可以通过`hexo new draft "草稿标题"`命令创建，存放在`source/_drafts`目录中。当草稿完成需要发布时，使用`hexo publish draft "草稿标题"`命令将其移动到`source/_posts`目录。

部署流程由`scripts/deploy.js`脚本自动化完成，包括验证服务器配置、构建静态文件、清理旧版本、备份当前版本、上传新文件和验证部署结果等步骤。回滚操作则由`scripts/rollback.js`脚本处理，当需要恢复到上一个版本时，该脚本会查找最新的备份文件并恢复。

## 模板文件详解

### post.md - 文章模板

`post.md`是创建新文章时使用的模板文件，位于`scaffolds`目录下。该模板定义了文章的基本Front Matter结构，包括标题、日期和标签字段。当执行`hexo new "文章标题"`命令时，Hexo会自动将`{{ title }}`替换为指定的标题，`{{ date }}`替换为当前日期，然后在`source/_posts`目录下创建相应的Markdown文件。

模板内容如下：
```
---
title: {{ title }}
date: {{ date }}
tags:
---
```

- [scaffolds/post.md](file://scaffolds/post.md#L1-L6)

### page.md - 页面模板

`page.md`是创建独立页面时使用的模板文件，同样位于`scaffolds`目录下。与文章模板相比，页面模板更为简洁，仅包含标题和日期字段。该模板用于创建如关于页面、归档页面等不需要分类和标签的独立页面。

模板内容如下：
```
---
title: {{ title }}
date: {{ date }}
---
```

**Section sources**

### draft.md - 草稿模板

`draft.md`是创建草稿时使用的模板文件，位于`scaffolds`目录下。该模板与文章模板类似，但不包含日期字段，允许作者在完成写作后再确定发布日期。草稿文件默认保存在`source/_drafts`目录中，直到通过`hexo publish`命令发布。

模板内容如下：
```
---
title: {{ title }}
tags:
---
```

## 文章创建与Front Matter

### 创建新文章

在`source/_posts`目录下创建新的Markdown文章是博客内容创作的核心。使用`hexo new "文章标题"`命令可以快速生成新文章文件，该命令会自动应用`post.md`模板并填充基本的Front Matter元数据。生成的文件名默认为`文章标题.md`，存储在`source/_posts`目录中。

### Front Matter元数据

Front Matter是位于Markdown文件顶部的YAML格式元数据块，用于定义文章的属性。以下是常用的Front Matter字段及其用途：

- **title**: 文章标题，显示在文章页眉和列表中
- **date**: 发布日期，格式为YYYY-MM-DD HH:MM:SS
- **tags**: 标签数组，用于文章分类和检索
- **categories**: 分类，用于组织文章结构
- **layout**: 布局类型，如post、page等
- **permalink**: 自定义文章链接

例如，一篇实际文章的Front Matter可能如下所示：
```
---
title: Git与GitLab的企业实战
date: 2021-03-08 21:35:56
tags:
- Git
- GitLab
- 工程化
categories: 前端开发
---
```

## Markdown语法规范

### 基本语法

博客支持标准的Markdown语法，包括标题、段落、列表、引用、代码块等。标题使用`#`符号，从`#`到`######`分别对应h1到h6标签。段落之间需要空行分隔。无序列表使用`-`、`+`或`*`，有序列表使用数字加点。

### 扩展特性

#### 代码高亮

通过三个反引号包围代码块，并指定语言类型，可以实现语法高亮。例如：
````
```javascript
function hello() {
    console.log("Hello World");
}
```
````

系统使用highlight.js进行代码高亮，支持多种编程语言。

#### 图片插入

图片插入使用`![alt文本](图片路径)`语法。图片可以是相对路径或绝对URL。建议将图片放在`source/images`目录下，并使用相对路径引用。

#### 表格

使用管道符`|`创建表格，例如：
```
| 命令 | 作用 |
|------|------|
| git init | 初始化本地库 |
| git add | 添加到暂存区 |
```

## 独立页面管理

### about.md - 关于页面

`about.md`是博客的关于页面，通常包含作者介绍、联系方式和个人经历。该页面使用`layout: about`指定特殊布局，并通过`permalink: about.html`设置固定链接。页面内容可以使用Markdown语法编写，支持标题、段落、列表和链接等元素。

示例内容：
```
---
title: 关于本人
layout: about
permalink: about.html
---
### 关于本人
[仓库](https://github.com/WQhuanm/Issue_Blog) 的其他内容
### [您好](https://giscus.app/zh-CN)
```

### friends.md - 友链页面

`friends.md`是博客的友链页面，用于展示友情链接。该页面使用`layout: friends`指定友链布局，并通过`permalink: friends.html`设置固定链接。友链配置也可以在`_config.kira.yml`文件中通过`friends`字段进行全局设置。

示例内容：
```
---
title: 我的朋友
layout: friends
permalink: friends.html
---

我的朋友
```

## 写作最佳实践

### SEO优化技巧

为了提高文章在搜索引擎中的可见性，建议遵循以下SEO优化技巧：

1. **关键词优化**：在标题、首段和正文中自然地包含相关关键词
2. **语义化标题**：使用h1-h6标签建立清晰的内容层次结构
3. **图片优化**：为所有图片添加描述性的alt文本
4. **内部链接**：在相关内容之间建立链接，提高页面权重
5. **外部链接**：引用权威来源，增加文章可信度

### 内容组织结构

良好的内容组织能提高读者的阅读体验：

1. **引言**：简要介绍文章主题和目的
2. **目录**：对于长篇文章，提供目录方便导航
3. **分节**：使用小标题将内容划分为逻辑段落
4. **总结**：在文章末尾总结要点，强化记忆
5. **行动号召**：鼓励读者评论、分享或进一步探索

### 多媒体资源引用

合理使用多媒体资源可以增强文章表现力：

1. **图片**：使用高质量图片说明概念，避免版权问题
2. **代码示例**：提供可运行的代码片段，帮助读者实践
3. **图表**：使用图表展示数据关系，提高信息传达效率
4. **视频**：嵌入相关视频教程，丰富学习体验

## 常见错误与修正

### Front Matter格式错误

常见错误包括YAML语法错误、缺少必要字段或字段名拼写错误。确保使用正确的缩进和冒号格式，所有列表项前有连字符。

**错误示例**：
```
---
title: 错误示例
tags: [标签1, 标签2]  # 错误：不应使用方括号
---
```

**正确示例**：
```
---
title: 正确示例
tags:
- 标签1
- 标签2
---
```

### 图片路径问题

图片无法显示通常是由于路径错误。确保图片路径正确，推荐使用相对路径。如果图片放在`source/images`目录下，引用时应为`/images/图片名.jpg`。

### 部署失败

部署失败可能由多种原因引起，包括服务器配置错误、网络问题或权限不足。检查`.env`文件中的服务器配置是否正确，确保有适当的SSH访问权限。
