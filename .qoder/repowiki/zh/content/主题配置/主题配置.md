## 配置文件概述

本博客项目采用双配置文件架构，通过分工协作实现灵活的外观和行为定制。主配置文件 `_config.yml` 负责控制Hexo框架的全局设置，而 `_config.kira.yml` 专门用于Kira主题的个性化选项。这种分离设计使得用户可以在不影响框架核心功能的前提下，自由定制主题的视觉风格和交互特性。

## Hexo全局配置详解

`_config.yml` 文件是Hexo框架的核心配置文件，定义了网站的基本信息、URL结构、目录设置、写作规则和部署方式等全局参数。该文件中的配置直接影响整个博客系统的运行机制。

### 站点基本信息
站点配置部分定义了博客的基本属性，包括标题、副标题、描述、关键字、作者和语言等元数据。这些信息不仅影响网站的显示效果，还对搜索引擎优化（SEO）起着重要作用。

### URL与目录设置
URL配置决定了文章链接的生成规则，支持自定义永久链接格式。目录设置则指定了源文件、输出文件、标签、归档等各类内容的存储路径，为博客的内容组织提供了灵活性。

### 写作与渲染配置
写作配置包含了文章默认布局、文件命名规则、代码高亮设置等。其中，`highlight.js` 和 `prismjs` 的配置允许用户自定义代码块的显示效果，如行号显示、自动检测语言等。

### 扩展与部署
扩展配置指定了当前使用的主题（kira），而部署配置则为空，表明项目使用自定义的部署脚本而非Hexo内置的部署方式。

## Kira主题配置详解

`_config.kira.yml` 文件是Kira主题的专属配置文件，提供了丰富的个性化选项，允许用户深度定制博客的外观和功能。

### 基础外观配置
```yaml
avatar: /img/头像.jpg
background:
  path: /img/codeBackground.png
  width: 1280
  height: 720
favicon:
  href: /img/favicon.jpg
  type: image/jpg
```
这些配置项分别定义了网站的头像、背景图片和favicon。背景图片同时作为博客的背景和文章的默认头图，实现了视觉上的统一性。

### 导航菜单配置
```yaml
menu:
  回到首页:
    - /
    - icon-home
  文章归档: 
    - /archives.html
    - icon-container
  关于本人:
    - /about.html
    - icon-user
  我的朋友:
    - /friends.html
    - icon-team
```
菜单配置采用层级结构，每个菜单项包含链接路径和图标名称。这种设计使得用户可以轻松添加、删除或重排导航项目。

### 社交链接配置
```yaml
social:
  QQ:
    - tencent://AddContact/?fromId=45&fromSubId=1&subcmd=all&uin=1964756692&website=www.oicqzone.com
    - icon-QQ
    - rgb(49, 174, 255)
    - rgba(49, 174, 255, .1)
  GitHub:
    - https://github.com/misaka12648
    - icon-github
    - rgb(25, 23, 23)
    - rgba(25, 23, 23, .15)
```
社交链接配置不仅包含URL和图标，还支持自定义颜色和悬停效果，实现了高度的视觉个性化。

### 配色方案配置
```yaml
color:
  first:
    r: 49
    g: 174
    b: 255
  second:
    r: 255
    g: 78
    b: 106
```
配色方案从first到seventh定义了7个优先级的颜色，支持RGB格式的精确颜色控制。这些颜色不仅用于主题色，还影响整个网站的彩虹配色效果。

### 评论系统集成
```yaml
giscus:
  active: true
  repo: misaka12648/misaka12648.github.io
  repoID: R_kgDOQAU8lQ
  category: Comments
  categoryID: DIC_kwDOQAU8lc4CwgBg
  mapping: pathname
  theme: light
  lang: zh-CN
```
Giscus评论系统配置实现了GitHub Discussions的无缝集成，支持多语言和主题切换，为博客提供了现代化的评论功能。

## 页面布局自定义

通过修改 `layout.ejs` 模板文件，用户可以自定义页面的整体布局结构。该文件位于 `themes/kira-custom/layout/` 目录下，是主题布局的主入口。

### 模板结构分析
```html
<!DOCTYPE html>
<html lang="<%- config.language %>">
<head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title><% if (title){ %> <%= title %> - <% } %> <%= config.title %></title>
    <%- css("css/layout") %>
    <!-- 各类CSS和JS资源引入 -->
</head>
<body>
    <div class="kira-background" style="background-image: url('<%= theme.background.path %>')"></div>
    <%- include("components/header") %>
    <div class="kira-body">
        <%- include("components/sidebar") %>
        <div class="kira-content">
            <div id="kira-top-header"></div>
            <div class="kira-main-content">
                <%- body %>
            </div>
        </div>
        <%- include("components/right-column") %>
    </div>
    <!-- 脚本引入 -->
</body>
</html>
```
该模板采用标准的HTML5结构，通过EJS模板语法动态插入配置值和组件。页面布局由背景层、头部、主体内容区（包含侧边栏和主内容区）和右侧栏组成。

### 关键布局特性
- **动态背景**：通过 `<%= theme.background.path %>` 动态插入背景图片路径
- **组件化结构**：使用 `<%- include() %>` 语法引入头部、侧边栏等组件，实现模块化设计
- **资源管理**：通过 `<%- css() %>` 和 `<%- js() %>` 语法统一管理CSS和JS资源
- **AI助手集成**：包含AI助手的配置脚本和样式链接，支持智能对话功能

## 样式覆盖与自定义

项目提供了多种方式实现样式覆盖和自定义，满足不同层次的定制需求。

### 自定义CSS文件
在 `source/css/` 目录下可以添加自定义CSS规则：
- `ai-assistant.styl`：AI助手组件的样式文件
- `github-markdown.css`：Markdown内容的显示样式

### 主题样式机制
Kira主题采用Stylus预处理器，通过 `color.styl` 文件实现动态配色：
```stylus
$color1 = rgb(hexo-config('color.first.r'), hexo-config('color.first.g'), hexo-config('color.first.b'))
```
该机制将 `_config.kira.yml` 中的配色配置转换为CSS变量，实现了配置驱动的样式系统。

### 样式覆盖流程
1. 在 `source/css/` 目录下创建新的Stylus或CSS文件
2. 在 `_config.kira.yml` 中的 `customStyles` 数组中添加文件路径
3. 重新生成博客以应用新样式

```yaml
customStyles:
  - style
  - custom
  - ai-assistant
```

```yaml

  enable: true
  silicon_flow:
    api_key: 
      - "sk-sbexyisgtctqiawgnvwlmlwssmdmtvsmqhnvoduczpxeoyow"
    model: "deepseek-ai/DeepSeek-V3.2"
  deepseek:
    api_key: "sk-4ab429704e324c5db0460cf03eee6285"
    model: "deepseek-chat"
```
支持硅基流动API和DeepSeek API双模式，提供故障转移能力。

### 组件样式修改
通过分析主题的Stylus文件结构，可以定位并修改特定组件的样式：
- `sidebar.styl`：侧边栏样式
- `post.styl`：文章页面样式
- `archive.styl`：归档页面样式

### 新页面模板创建
1. 在 `themes/kira-custom/layout/` 目录下创建新的EJS模板文件
2. 复制现有模板结构并进行修改
3. 在Hexo配置中指定新模板的使用规则

## 配置生效与预览

### 配置修改流程
1. 修改 `_config.yml` 或 `_config.kira.yml` 文件
2. 如需修改布局，编辑 `layout.ejs` 文件
3. 如需添加自定义样式，创建或修改 `source/css/` 目录下的CSS文件

### 本地预览
```bash
cd myblog
hexo server
```
博客将在 `http://localhost:4000` 启动，实时预览配置修改效果。

### 部署前的重要步骤
由于 `layout.ejs` 文件被封装在 `node_modules` 中，每次修改后需要手动执行：
```bash
cp themes/kira-custom/layout/layout.ejs node_modules/hexo-theme-kira/layout/layout.ejs
```
此步骤确保自定义布局文件被正确应用。

### 构建与部署
```bash
npm run deploy
```
使用项目内置的自动化部署脚本，支持一键部署到服务器。
